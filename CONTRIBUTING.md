# Contributing to MyKid

Thank you for your interest in contributing to MyKid! This document provides guidelines for reporting bugs, suggesting features, and submitting pull requests.

## Reporting bugs

- Use the [GitHub Issues](https://github.com/scroodge/MyKid/issues) to report bugs.
- Describe the issue clearly: what you expected vs what happened.
- Include your environment: Flutter version, device/OS, Supabase and Immich versions if relevant.

## Suggesting features

- Open an issue with the `enhancement` label.
- Describe the use case and why it would benefit other users.

## Pull requests

1. Fork the repository and create a branch from `main`.
2. Make your changes. Follow the existing code style.
3. Ensure tests pass (if applicable) and the app runs.
4. Update documentation if needed (README, `docs/` â€” e.g. when adding a new AI provider or feature).
5. Submit a PR with a clear description of the change.
